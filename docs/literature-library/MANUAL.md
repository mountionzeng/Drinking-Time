# 文学库 Manual

> 文学库 = 一堆「文学家声音」条目。剧本 Agent 按当前用户的情绪流动，从库里取若干把声音作**共鸣参照**，注入剧本生成。
> 它与美术库 `docs/style-library/` 同构：同一套加载底座、同一套落点字段，区别只在 DNA 字段是「文学」而非「视觉」。

## 一条目 = 一把声音

- 一个 `entries/<id>.yaml` = 一位文学家 / 一把可辨识的声音。
- 复制 `_TEMPLATE.yaml` 到 `entries/<id>.yaml` 开始填。
- `id` 是英文小写连字符 slug，系统引用用，定了别改。

## 核心守则：共鸣，不是代笔

文学库的作用是**借文学家的视角和声音，照见用户自己的思绪**，让剧本与用户的情绪流动产生共鸣。它**不是**：

- 不是把 `signature_lines`（代表句）原样塞进剧本——那是抄。代表句只是「这把声音长什么样」的参照。
- 不替用户拔高、升华、加道德评判。
- 不伪造用户没说过的事实或原话。

## 字段怎么填

- `viewpoint`（观点）：这把声音如何看待人生与情感。最该点名的一栏。
- `voice`（声音）：句法、语气、节奏、密度——读一段就能认出来的那种特征。
- `themes` / `devices`：母题与手法。
- `signature_lines`：代表句，作参照。
- `negative`：这把声音**最不该**被写成什么（和正面同等重要，防跑偏）。
- 落点 `emotion_fit` / `theme_fit` / `affinity`：共鸣排序的先验。只是冷启动权重，用户真实的情绪流动会覆盖它。

## status

- `draft`：在养、还没上线，不进共鸣候选。
- `active`：进共鸣候选。坏条目（缺 id/name、YAML 解析失败）会被加载器告警跳过，不会让整库崩。
