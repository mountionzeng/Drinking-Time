# 流派库 · 搭建说明书

> 库是小酌出图的**护城河**：6 张草稿，就是从这个库里、按「落点」给每个用户排出来的 6 个流派。
> 一条目 = 一个美术流派 / 表达形式 = 把系统**已经认得**的 prompt 槽位，由你这个「有眼光的人」预先手工填好一捆。
> 你不是在学新东西，是在把脑子里的判断填进系统已有的格子。

---

## 0. 文件结构

```
docs/style-library/
├─ MANUAL.md            ← 你在看的这份说明书
├─ _TEMPLATE.yaml       ← 复制它开始写新条目
├─ entries/             ← 一条目一个文件（现 15 条）
│  ├─ rembrandt-oil.yaml     ← 种子·active
│  ├─ shinkai-light.yaml     ← 种子·active
│  ├─ song-ink.yaml          ← 种子·active
│  └─ …                      ← 另 12 条 draft，待你出图校准后转 active
└─ refs/                ← 范例图放这（条目里用相对路径引用，现为占位待补真图）
```

> 最终库怎么被系统加载、放哪个目录，是 planning 的事。现在这里是库内容的**唯一真相源**，先安心攒。

## 1. 加一条的最快路径

1. 复制 `_TEMPLATE.yaml` → `entries/<id>.yaml`
2. 填 `id` / `name` / `one_liner`
3. 填**视觉 DNA** 八栏（style / palette / light / composition / material / era_culture / signature / negative）
4. 用产品出一张草稿，看**认不认得出**这个流派；不像就改词（见 §3）
5. 标**落点**（emotion_fit / theme_fit / affinity）
6. 满意了把 `status` 改成 `active`，它就进 6 候选轮换

---

## 2. 字段怎么填（带「为什么」）

| 字段 | 填什么 | 为什么 |
|---|---|---|
| `id` | 英文小写连字符 slug | 系统稳定引用，定了别改 |
| `name` | 中文展示名 | 给用户/你看 |
| `one_liner` | 一句人话气质 | 给小酌当口播，也帮你自己锚定 |
| `style` | 媒介/技法/流派名/**点名** | 最关键。"in the manner of X" 是最稳的拉风格杠杆 |
| `palette` | 色调清单 | 色是流派的半条命 |
| `light` | 光的签名 | 光往往比内容更定流派 |
| `composition` | 景别/主体位置/留白 | 决定"气" |
| `material` | 笔触/颗粒/纹理 | 决定"质感"，防塑料感 |
| `era_culture` | 时空坐标 | 把模型拉进对的参照系 |
| `signature` | ★一眼签名 | 低分辨率草稿里也得认得出的那个点 |
| `negative` | 最怕变成什么 | 和正面一样重要 |
| `emotion_fit` / `theme_fit` | 适配情绪/题材 | 和「当前故事情绪」匹配排序 |
| `affinity` | 职业/年龄/五行权重 0–3 | 冷启动先验（见 §5） |
| `references` | 1–3 张范例图 | 给系统做风格条件，给你校准眼睛 |

---

## 3. 写好一条的 7 条手艺规矩

1. **写「画得出来的词」，不是形容词。** 给媒介/技法/光的**名字**（厚涂油画 / 侧逆光 / 胶片颗粒），别写"很美很有感觉"——模型认前者。
2. **让它在草稿就认得出。** 每条要有一个一眼 `signature`（光、色、轮廓三者之一够强），低分辨率草稿才"显学派"——这正是你「便宜草稿也得好看」的关键。
3. **用名字锚定。** 流派 / 艺术家 / 电影，点名最稳——这事你比谁都强。
4. **负面和正面一样重要。** 写清它最容易翻车成什么（伦勃朗光最怕平光+霓虹）。
5. **一条目一学派，别在条目里混搭。** 混搭是运行时合成缝的活；库里每条要纯，6 张才拉得开。
6. **配 1–3 张范例图。** 既给系统做风格条件，也给你自己校准。
7. **标「落点」要诚实。** 真有把握才标 affinity，不确定就留空（见 §5）。

---

## 4. 怎么让 6 张「拉得开」——库的光谱

你不是在收集"好看的图"，是在收集**互相不一样的好看**。整库按这几条轴铺开，别让条目挤在一个角：

- **媒介**：油画 / 水彩 / 水墨 / 胶片摄影 / 3D / 版画 / 拼贴 …
- **色温**：暖 ↔ 冷；高饱和 ↔ 低饱和灰
- **年代**：古典 ↔ 当代 ↔ 未来
- **繁简**：满构图厚信息 ↔ 极简留白
- **情绪温度**：炽烈 ↔ 温吞 ↔ 清冷

**自检**：随便抽 6 条放一起，像不像 6 个不同的人？像就对了；像同一个人换 6 件衣服，就太挤。

---

## 5. 落点 & 亲和：冷启动用的先验

- `emotion_fit` / `theme_fit`：这流派适合接什么情绪/题材——排序时和**当前故事情绪**匹配。
- `affinity`：职业/年龄/五行 的亲和权重 0–3。**这只是开局先验**：新用户一张图都没划过时拿它排序；用户一旦开始多选/略过，真实审美信号（`imageSignals` / `semanticAnnotations`）就接管，先验权重退场。
- **守则：宁可少标、别硬标。** 你不确定某流派配什么人，就**留空**——硬标 = 制造刻板印象，还会污染冷启动。

---

## 6. 工作流：用「出图」养库，不是改代码

```
白话写一稿 → 出草稿 → 认不认得出 → 改词到稳定 → 标落点 → status: active
```

库越喂越准，就是你的审美护城河在产品里一点点长出来。

> **Agent-native：** 以后小酌也能读写这个库——它能在对话里帮你提议新流派、从用户反馈里建议补条目。**你做得到的，它也做得到。**

---

## 7. 质量自检清单（一条上线前过一遍）

- [ ] `signature` 这栏，低分辨率草稿里也认得出吗？
- [ ] `style` 里有没有**点名**（流派 / 艺术家 / 电影）？
- [ ] `negative` 写了它最容易翻车成什么吗？
- [ ] 词是"画得出来的"（媒介/技法/光），不是形容词？
- [ ] 和库里已有条目够不够**不一样**（§4 光谱）？
- [ ] `affinity` 是真有把握才标，不是硬凑的？

---

## 8. 一条守则警告（最重要）

**流派只换「怎么好看」，不换「发生了什么」。**

小酌现有守则——如实镜像用户材料、不为戏剧放大、不翻负、不正向失真——仍然管着每一次出图。伦勃朗光不能把一个平静的下午画成悲剧。**库是审美透镜，不是情绪滤镜。**
