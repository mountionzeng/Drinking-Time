# 出图简化：一张漂亮、可复用的图，作为故事材料与视频关键帧

## Summary

把出图从"一次出 4~6 张候选、又贵又慢、失败还消失"，改成"点一次『把这一刻画出来』→ 出**一张**图 → 满意（右划/选 OK）就留下并继续聊天，不满意（划走）就再出一张，**直到满意为止**"。满意的那张直接落为镜头/卡片下的**故事材料（关键帧 + 提示词 + 美术配方）**、可复用、不丢。美术定调降级为轻量自动继承，出图按需逐镜，失败不清空。材料做成"视频就绪"的形态，但视频合成本身不在本次。

---

## Problem Frame

用户喜欢"把这一刻画出来"这个简单入口，但当前出图链路又贵又不顺，且出来的图留不住：

- **贵**：点一次"生成画面"底下并发出 4~6 张 Midjourney 候选（`generateArtDirectionCandidates`），是单图成本的数倍；带参考图还各过一次 LLM 视觉分析；小酌每条消息 system prompt 塞了卡片+剧本+镜头表+图片资产摘要，上下文越长越贵。用户"以为出一张图"，系统其实做了一大串。
- **不顺**：点"生成画面"等很久（多候选 + 上游慢）；点"提炼故事视觉配方"后**图没了**——已诊断为外部 302.AI 无可用模型/Midjourney 超时导致候选全失败，叠加 studio "生成中"状态先清空旧候选、失败又不回滚，于是空屏。
- **留不住**：图与故事材料的关系不够稳固，用户希望一张满意的图能**确定地挂在镜头/卡片下、后续复用**，并最终作为视频的关键帧。

用户明确：**不要一次出那么多图，一次一张就好，但这张要够漂亮、且后续用得了。**

---

## Actors

- A1. 用户：项目所有者，在聊天里点"把这一刻画出来"，希望简单、出一张好图、能留用。
- A2. 小酌（creation 聊天 Agent）：根据当前对话/素材出图、按当前故事归属。
- A3. 美术定调（轻量）：提供"本故事视觉风格"，让单张图够漂亮、风格一致——自动继承，不强迫用户走探索 studio。

---

## Key Flows

- F1. 把这一刻画出来（核心，一次一张、划走再来、直到满意）
  - **Trigger:** 用户在聊天里点"把这一刻画出来"
  - **Actors:** A1, A2, A3
  - **Steps:** 取当前对话/素材 + 当前故事的视觉配方（若已锁定，自动继承）→ 出**一张**图给用户看 → 用户二选一：
    - **满意（右划 / 选 OK）** → 这张收下为该镜头/卡片的关键帧（故事材料），**循环结束，回到正常聊天**；
    - **不满意（划走 / 左划）** → 记一个否定信号，**再出下一张** → 重复，**直到用户满意为止**
  - **Outcome:** 用户只在满意时停下；满意的那张明确归属到某镜头/卡片，可复用、不丢
  - **Covered by:** R1, R2, R3, R5, R9

- F2. 改这张（在已收下的图上微调）
  - **Trigger:** 用户对已收下的关键帧想整图微调（"改暖一点"等）
  - **Actors:** A1, A2
  - **Steps:** 对当前关键帧做整图修改（reviseImage）→ 新版进待确认 → 满意则替换为该镜关键帧，旧版进历史
  - **Outcome:** 已有材料上的迭代，不另起一堆
  - **Covered by:** R4, R6

- F3. 出图失败时
  - **Trigger:** 出图服务（302.AI/Midjourney）不可用或超时
  - **Actors:** A1, A2
  - **Steps:** 明确提示"出图服务暂时不可用，稍后再试" → **保留之前的图，不清空**
  - **Outcome:** 用户知道发生了什么，已有材料不丢
  - **Covered by:** R7

## Requirements

**简单出图（一次一张、划走再来、直到满意）**
- R1. "把这一刻画出来"是简单主入口：点一次出**一张**图（不是候选集），基于当前对话/素材生成，给用户看。
- R2. 单张图的质量靠**自动继承的美术配方**保证（当前故事若已锁定视觉配方则继承；未锁定用合理默认），让单张就够漂亮、风格一致——用户**不必**走 6 张候选的探索 studio。
- R3. 出图按需、逐镜：不一次性为所有镜头出图，用哪镜出哪镜，省掉"出一堆又丢弃"的浪费。
- R9. **划走再来、直到满意**：用户对当前这张二选一——满意（右划/选 OK）则收下、循环结束回到聊天；不满意（划走/左划）则记否定信号并**再出下一张**，重复直到满意。每次只出一张、只展示当前一张，不堆候选。被划走的旧版可进历史，不直接消失。

**图作为可复用的故事材料 / 关键帧**
- R4. 每个镜头/卡片下有一条"画面材料"：选定的**关键帧图 + 提示词 + 美术配方**（外加预留的视频参数位）。
- R5. 一张满意的图**确定地归属到某镜头/卡片**（按当前故事独立，沿用 per-story），可复用、刷新/切换不丢、不串故事。
- R6. 可对某镜的图迭代：再出一张（旧版进历史）、整图修改、收下为主图（关键帧）。

**健壮性**
- R7. 出图失败/超时时：明确提示"出图服务暂时不可用"，**保留之前的图、不清空**，不让用户误以为"图没了"。

**视频就绪（只做数据形态，不做合成）**
- R8. "画面材料"的结构是"视频就绪"的——关键帧 + 提示词（+ 预留视频参数）足以作为后续视频合成的输入；本次**只**确保数据形态到位，不做视频合成/导出。

## Acceptance Examples

- AE1. **Covers R1, R2.** 当前故事已锁定视觉配方，用户点一次"把这一刻画出来"：系统出**一张**继承该配方的图（不是 4~6 张），风格与本故事一致。
- AE5. **Covers R9.** 用户点"把这一刻画出来"出第 1 张 → 划走 → 系统出第 2 张（同一镜头、记了第 1 张的否定信号）→ 划走 → 出第 3 张 → 用户右划满意：第 3 张收下为该镜关键帧，循环结束、回到聊天；全程任一时刻屏上只有当前一张，不堆候选。
- AE2. **Covers R5.** 用户为镜头 SH02 出了一张满意的图、收下为主图；刷新页面 / 切到别的故事再切回来：该图仍挂在 SH02 下、不丢、不串到别的故事。
- AE3. **Covers R7.** 出图服务返回不可用/超时：界面提示"出图服务暂时不可用，稍后再试"，且**之前已有的图仍在**，没有被清空。
- AE4. **Covers R3.** 一个故事有 5 个镜头，用户只为其中 2 个镜头点了"画出来"：系统只出了这 2 张图，没有为另外 3 个镜头预渲染浪费。

## Success Criteria

- 用户点一次"把这一刻画出来"，**几秒到十几秒内拿到一张够漂亮的图**（而非等多候选），且这张图明确属于某镜头/卡片、后续能复用。
- 出图相关的 token/$ 花费**明显下降**（不再一次出 4~6 张、不再为不用的镜头预渲染）。
- 出图失败时用户**不再看到"图莫名没了"**，而是清楚的提示 + 旧图保留。
- 交接给 ce-plan 时不需要再发明产品行为：入口、单张策略、材料结构、归属、失败行为、视频就绪边界均已定义。

## Scope Boundaries

### Deferred for later
- **真视频合成 / 导出**：本次只把"画面材料"做成视频就绪的数据形态；视频合成是单独的大项目（已搁置）。
- 美术定调的"探索/收敛 studio"（6 张候选挑选）：保留为可选的进阶路径或后续，不作为主路径——主路径是"一次一张 + 自动继承配方"。

### Outside this product's identity
- 解决 302.AI/Midjourney 外部故障、或接入/更换出图模型供应商——属外部/基础设施，不是本次产品决策。
- prod 数据迁移（另有 `docs/prod-storyid-backfill-runbook.md`）。

---

## Key Decisions

- **一次一张、自动继承配方** 取代"多候选探索 studio"作为主路径：用户要的是简单 + 一张漂亮可复用的图，不是挑图工作量。质量靠继承的视觉配方保证，而非让用户从 6 张里选。
- **图 = 故事材料（关键帧 + 提示词 + 配方）**，挂在镜头/卡片下：这是用户"图对应到卡片""图+提示词=视频材料"心智模型的落地，也是通往视频的桥。
- **按需逐镜渲染**（吸收路线 C 省钱思路）：不一次性为所有镜头出图，砍掉"出一堆又丢弃"的浪费。
- **失败不清空**：把"图没了"从外部故障的连带伤害里救出来——保留旧图 + 明确提示。
- 视频合成**明确不在本次**：只做数据形态，避免范围爆炸。

---

## Dependencies / Assumptions

- 已落地的地基本次直接复用：故事为唯一单位、镜头/图片按 storyId 归属（per-story 独立）、图片资产层（`server/services/imageAssets.ts`）、`renderViaGate` 出图唯一必经点、`creationGoal` 目标注入。
- "美术配方/视觉配方锁定"已存在（前端有"锁定为本故事视觉配方""后续镜头默认继承"），本次把"单张自动继承"这条接紧；配方的产生方式（是否仍可用现有 studio 生成、还是给个轻量默认）留规划期定。
- "把这一刻画出来"入口已存在（移动端 `MobileChatContext` 的手动"画出来" + 故事页 launcher），本次统一/简化其行为为"一次一张 + 归属材料"。
- 外部出图 API（302.AI/Midjourney）的可用性不在本次掌控；本次只保证失败时的体验，不修上游。
