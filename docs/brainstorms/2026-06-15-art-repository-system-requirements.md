# 美术参考库系统需求文档

## 问题背景

当前系统生成的图片质量不稳定，原因是缺乏统一的、可学习的美术标准。用户已积累了 99 张精选参考素材，希望系统能够：
1. 从这些参考中**自动学习**美术指导
2. 每次生成图片时**动态应用**这些指导
3. **根据用户反馈**持续优化生成质量

## 核心目标

**确保所有生成的图片都符合用户的审美标准，通过系统反馈循环来持续改进。**

---

## 系统设计

### 第一部分：参考库与特征提取

#### 1.1 参考库结构

```
art-repository/
├── references/              # 99 张精选参考图（1024×1024）
├── features-cache.json      # 自动提取的特征缓存
├── rules/
│   └── global-rules.md      # 全局美术规则（用户编写）
└── README.md
```

#### 1.2 特征缓存内容

系统用 Vision API 自动分析每张参考图，提取以下特征：

```json
{
  "某张参考.jpg": {
    "artStyle": "水彩插画",
    "artistReference": "宫崎骏风格",
    "dominantColors": ["金色(暖亮)", "褐色(暖中)"],
    "colorTone": "warm-light",
    "lightingCharacter": "侧光，主光源清晰",
    "mood": ["温暖", "安静", "思考"],
    "composition": "人物中景，背景虚化",
    "materials": ["纸质", "布料", "皮肤"],
    "cameraAngle": "略微俯视",
    "visualDescription": "温暖的侧光照亮了房间..."
  }
}
```

**关键特征说明：**
- **artStyle（美术流派）**：水彩、油画、素描、插画等 — **最高优先级**
- **dominantColors（主要色彩）**：第二优先级
- **materials（物体材质）**：从参考图和用户描述中提取
- **其他特征**：支撑信息

#### 1.3 全局美术规则

用户在 `rules/global-rules.md` 中编写美术原则，系统在生成时应用：

```markdown
## 色彩与光线
- 色调倾向于暖色，避免冷冰冰的蓝调
- 光线必须有明确的主光源，避免均匀平面光

## 避免事项
- 避免过度饱和的色彩
- 避免硬朗的摄影棚灯
```

### 第二部分：生成流程与动态提取

#### 2.1 生成触发点

用户在聊天界面点击"把这一刻画出来"按钮，系统执行以下流程：

#### 2.2 核心流程

```
┌─ 用户点击"把这一刻画出来"
│
├─ [收集信息]
│  ├─ 当前故事内容（镜头描述、人物、物体等）
│  ├─ 用户的文字描述（可选）
│  └─ 用户上传的参考图（可选）
│
├─ [提取维度信息]
│  ├─ 美术流派期望（从描述中识别"手绘感"→水彩、插画）
│  ├─ 颜色信号（"夜晚"→深色；"阳光"→金色）
│  ├─ 材质信息（从用户描述中提取）
│  └─ 情感/场景信息（思考、温暖、室内等）
│
├─ [参考库匹配（多维，优先级从高到低）]
│  ① 美术流派匹配（50%权重）
│     筛选符合期望流派的参考
│  ② 颜色匹配（30%权重）
│     在筛选结果中找颜色相近的
│  ③ 情感/场景匹配（15%权重）
│  ④ 材质匹配（5%权重）
│
├─ [融合指导信息]
│  ├─ 美术风格信息
│  ├─ 参考图的特征描述
│  ├─ 全局美术规则
│  ├─ 用户满意偏好（如果有）
│  └─ 避免之前不满意的配置
│
├─ [生成提示词]
│  prompt = "【美术流派 + 艺术家】，原始描述，【颜色指导】，【材质指导】，【规则】"
│
└─ [生成图片]
   调用 generateForMobile 出图
```

#### 2.3 用户反馈循环

生成完图片后，用户看到三个选项：

**选项 A：[满意]**
- 系统记录此生成配置到"满意偏好库"
- 下次遇到类似场景优先使用此配置
- 同一配置出现 3 次满意则计为 1 条偏好记录

**选项 B：[不满意/划走]**
- 出现文本框 + 硬编码选项
- 硬编码选项：
  - □ 色调太冷/太暖
  - □ 美术风格不对（比如"太逼真，想要手绘感"）
  - □ 材质不对
  - □ 人物表情/姿态不对
  - □ 构图不满意
  - □ 其他：________
- 用户可选择 + 自由输入

**系统反应：**
1. LLM 理解用户的反馈（解析用户说的是什么维度的问题）
2. 存储反馈到"不满意反馈库"（用来学习什么时候不要用某种配置）
3. **立即重新生成**（基于理解调整参考库的应用或规则）
4. 用户可继续反馈或满意确认

#### 2.4 偏好库管理

**满意偏好库** - 记录用户喜欢的配置

```json
{
  "userId:projectId": {
    "preferences": [
      {
        "sceneFingerprint": {
          "timeOfDay": "夜晚",
          "location": "房间",
          "pose": "坐着",
          "emotion": "思考",
          "objects": ["书桌", "椅子"],
          "materials": ["木质"]
        },
        "artStyle": "水彩插画",
        "usedReferences": ["ref1.jpg", "ref2.jpg"],
        "colorScheme": "warm-dark",
        "appliedRules": ["rule1"],
        "generatedPrompt": "...",
        "likeCount": 3,
        "createdAt": "2026-06-15"
      }
    ]
  }
}
```

**不满意反馈库** - 记录用户不喜欢的原因

```json
{
  "userId:projectId": {
    "feedbacks": [
      {
        "sceneFingerprint": { /* 同上 */ },
        "usedConfiguration": { /* 生成时用的配置 */ },
        "userFeedback": "色调太冷了",
        "systemUnderstanding": {
          "dimension": "color",
          "problem": "cold",
          "shouldBe": "warm"
        },
        "createdAt": "2026-06-15"
      }
    ]
  }
}
```

## 关键设计决策

### 1. Vision-First，无人工标签
- 参考库的特征完全由 Vision API 自动识别
- 用户不需要逐张标记参考图
- 一次性脚本分析 99 张图，后续维护成本低

### 2. 美术流派最优先
- artStyle（美术流派）权重 50%，最高
- colorTone 权重 30%，次高
- 其他维度权重更低
- **理由**：美术风格决定了整体视觉表现，是最关键的

### 3. 细粒度偏好匹配
- 按镜头的具体内容（时间、地点、人物、物体、情感等）匹配
- 不是简单的场景分类，而是多维特征组合
- 部分匹配也可用（比如"夜晚+房间+坐着+思考"缺少一个维度，仍用最接近的满意偏好）

### 4. 双向学习
- **满意反馈**：学习用户喜欢什么配置
- **不满意反馈**：学习用户不想要什么，同时收集改进信息
- 两者都要保存，形成完整的学习闭环

### 5. 材质信息的两个来源
- **参考库**：从图片中自动识别
- **用户描述**：从文字中提取（"木制书桌"→"木质"）

- [ ] 用户反馈能被正确理解和记录
- [ ] 系统能根据用户偏好优先生成满意的配置
- [ ] 系统能从不满意反馈中学习并改进
