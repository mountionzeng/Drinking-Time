# Analysis Page Frontend Architecture Restructure

**Date:** 2026-05-09
**Status:** Draft
**Scope:** Standard

---

## Problem Statement

Drinking Time 的 `/analysis` 页面当前存在以下体验问题：

1. **入口混乱**：新用户打开页面后面对多个空面板，不知道第一步该做什么。
2. **层级扁平**：DropZone（输入）、TemplateDraft（中间产物）、ShotTable（产出）、StoryAgentChat（对话引导）等模块在页面上权重相同，没有主次之分。
3. **双路径混淆**：素材驱动路径（上传 -> 分析 -> 镜头表）和故事驱动路径（对话 -> 故事卡片 -> 剧本）共存，但两者的关系、入口和交汇点不清晰。
4. **状态不透明**：用户看不到"分析有没有跑过""素材有没有上传过""镜头表为什么是空的"等状态信息。
5. **辅助功能占主位**：TweaksDock、StageAtlas、纳音五行详细信息等装饰性/辅助性功能占据了核心位置，分散注意力。

## Users

影视视觉开发团队成员：整理参考图、剧本、brief，使用 AI 辅助生成镜头表和创作提示词。当前阶段以创始人自用 + 原型验证为主。

## Success Criteria

1. 新用户打开页面后，5 秒内知道第一步该做什么。
2. 两条创作路径（素材驱动 / 故事驱动）各自有清晰的入口和操作流程。
3. 输入、加工、产出三个层级在视觉上有明确的区分。
4. 完成一个操作后，用户能看到下一步的引导或状态变化。
5. 切换五行主题后，页面整体色调有明显可感知的变化（不只是 accent 色变）。

## Key Decisions

### 1. 架构方案：双入口引导页 + 左中右工作台面板

页面分为两个阶段：

**引导阶段**（新项目 / 无素材无卡片时）：
- 页面中心展示两个入口卡片，简洁明了：
  - "上传素材开始" -> 进入素材驱动路径
  - "聊一段故事开始" -> 进入故事驱动路径
- 底部说明文案："两条路径最终都会汇聚到镜头表，你也可以两个都用"
- 如果已有其他项目，侧边或顶部提供项目切换

**工作台阶段**（有素材或有故事卡片后）：
- 左中右三栏面板布局，面板可折叠
- 左面板 = 输入区
- 中面板 = 加工区
- 右面板 = 产出区

### 2. 两条路径并行，各自清晰

- **素材驱动路径**：DropZone（左面板）-> TemplateDraft（中面板）-> ShotTable + PromptDistill（右面板）
- **故事驱动路径**：StoryAgentChat（左面板）-> StoryCardsBoard（中面板）-> 生成剧本 / 转为镜头（右面板）
- 左面板用标签页切换两条路径的入口
- 中面板根据当前激活路径显示对应内容
- 右面板为统一产出区，两条路径的产出汇聚于此

### 3. 模块位置分配

| 模块 | 位置 | 显示策略 | 理由 |
|------|------|----------|------|
| DropZone | 左面板 Tab 1 | 常驻 | 核心输入入口 |
| StoryAgentChat | 左面板 Tab 2 | 常驻 | 核心输入入口 |
| TemplateDraft | 中面板 | 有分析数据时显示 | 素材路径的中间产物 |
| StoryCardsBoard | 中面板 | 有故事卡片时显示 | 故事路径的中间产物 |
| ShotTable | 右面板（上） | 常驻（空状态提示） | 核心产出 |
| PromptDistill | 右面板（下） | 有分析数据时显示 | 核心产出 |
| Timeline | 底部抽屉 | 点击触发 | 辅助参考，低频查看 |
| TweaksDock | 顶栏齿轮菜单 | 点击展开 | 装饰性设置，极低频 |
| StageAtlas | 折叠面板或设置区 | 默认折叠 | 展示性，非操作核心 |
| TopBar 纳音信息 | 顶栏小按钮 | 保留但精简 | 品牌元素但降低信息密度 |
| ProfileRailDrawer | 顶栏头像下拉 | 点击展开 | 低频操作 |
| BeverageAmbience | 背景层 | 保留 | 品牌氛围，不占交互空间 |

### 4. 首次打开引导设计

用户第一次打开页面时的操作顺序：
1. 看到页面中心的两个入口卡片
2. 选择一条路径开始（上传素材 或 聊故事）
3. 完成输入后，页面自动切换到工作台视图
4. 中面板显示加工中的内容（TemplateDraft 或 StoryCardsBoard）
5. 右面板随着分析完成逐步填充产出内容

### 5. TopBar 精简

当前 TopBar 同时显示：日期、农历、干支、纳音、镜头状态图例。
精简为：
- 保留：项目名 + 项目切换 + 纳音主题小按钮 + 用户头像
- 收起：农历/干支详细信息移入纳音弹窗内
- 移除或收起：镜头状态图例（放到 ShotTable 组件内部即可）

### 6. 纳音五行主题深化 —— 增大视觉差异

**现状问题**：切换五行后只有 accent 色变了，整体页面 90% 面积（白底、面板、文字）没变，视觉差异太小。具体原因：
- 金(H78) 和 水(H70) 色相只差 8 度，几乎分不开
- 水(C:0.08) 和 土(C:0.08) 饱和度太低，看起来灰蒙蒙
- 背景图 opacity 仅 0.18，几乎看不到

**改进方向**：

**a) 扩大色彩差异**

| 五行 | 饮品 | 色调方向 | 当前问题 | 改进 |
|------|------|----------|----------|------|
| 金 | 啤酒 | 琥珀金、麦芽黄 | OK | 保持，微调更金 |
| 木 | 龙井 | 嫩绿、翠绿 | OK | 保持 |
| 水 | 椰汁 | 乳白、椰壳棕 | 和金太像 | 色相拉远到暖白/象牙方向，提高饱和度 |
| 火 | 大红袍 | 深红、琥珀红 | OK | 保持 |
| 土 | 咖啡 | 深棕、焦糖 | 饱和度太低 | 提高饱和度和明度对比 |

**b) 扩展 CSS 变量覆盖范围**

目前只覆盖 accent 色系（4 个变量）。增加以下变量让整个页面色调跟着变：
- `--nayin-surface`：面板/卡片背景色（从纯白微调为偏暖/偏冷/偏绿等）
- `--nayin-surface-dim`：次级背景
- `--nayin-text-subtle`：辅助文字颜色
- `--nayin-border`：边框色调
- `--nayin-bg-gradient`：页面整体背景渐变

**c) 提高背景氛围可见度**

- 背景图 opacity 从 0.18 提高到 0.25-0.35
- 粒子动画增大尺寸和不透明度
- 每种五行可以有不同的面板边框风格（如金偏圆角、木偏直角、火偏锐利等）

**d) 保持切换机制不变**

- 默认根据当日纳音自动切换（已实现）
- 用户可手动预览切换（已实现）
- 全屏倾倒过渡动画（已实现）
- 切换后应用新的扩展变量集

## Scope Boundaries

### In scope
- `/analysis` 页面的布局重组和组件重新排列
- 引导页状态的新增
- TopBar 精简
- 辅助模块降级（TweaksDock、StageAtlas 收进菜单/折叠区）
- 面板折叠交互
- 纳音五行主题色深化：扩大五种主题的视觉差异

### Deferred for later
- Story Agent 的 prompt 策略和对话逻辑优化
- AI/LLM 后端架构调整
- 移动端响应式适配
- Creation Engine（节点式创作界面）
- 数据库 schema 变更
- 素材驱动和故事驱动路径之间的数据交叉引用

### Out of scope
- 新增功能模块
- 后端 API 变更
- 部署架构

## Dependencies / Assumptions

- 现有组件（DropZone、StoryAgentChat、ShotTable 等）的内部逻辑不需要改变，只改它们在页面上的位置和显示条件
- `useAnalysisWorkspace` hook 可能需要拆分或调整，以支持面板折叠状态和引导页/工作台的切换
- 项目已使用 `react-resizable-panels` 依赖，可以直接用于面板布局
- Tailwind CSS 4 + shadcn/ui 组件库已就位
