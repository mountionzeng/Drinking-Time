# Inline Selection Edit — Requirements

**Date:** 2026-05-20
**Status:** Ready for planning
**Scope:** Standard

## Problem

Users generate story content (cards, scripts, shots) through conversation with the Agent. When they want to adjust specific wording — a single phrase that feels wrong, or a sentence they'd like expanded — they currently have to either:

1. Manually edit via `contentEditable` (Agent doesn't learn from it), or
2. Describe the change in chat without precise reference (Agent might rewrite too much)

Neither approach gives the Agent a clear "original text → user intent → corrected text" signal, which is the highest-quality style feedback possible.

## Solution

Let users **select any text** across all content areas, **quote it into the chat**, tell the Agent how to change it, and have the Agent **apply the change precisely** to only the selected portion. Each correction feeds into the existing `semanticAnnotation` pipeline so future generation reflects the user's style.

## User Flow

1. User selects text anywhere (card content, script scene, shot field, or Agent chat reply)
2. A **quote block** appears above the chat input area, showing:
   - Source label (e.g., "Card 3" / "Scene 2" / "Agent reply")
   - Selected text (truncated if long)
   - Dismiss button (x)
3. User types their instruction in the chat input (e.g., "换成更冷的氛围" / "这个表述我很喜欢" / "在后面加一句关于光线的描写")
4. Agent replies in chat confirming the change, showing before/after
5. The **original content updates in place** — only the selected portion changes
6. A `semanticAnnotation` is generated from this correction, recording the factual change and inferred style preference

## Functional Requirements

### Selection Capture

- Use browser-native `Selection API` to detect text selection across all content areas
- Track selection **source metadata**: which entity (card ID, script scene index, shot ID, or chat message index) and the character offset range within that entity
- Selection highlight persists while the quote block is active (clear on dismiss or after edit applied)

### Quote Block in Chat Input

- Appears above the text input when a selection is active
- Shows: source label + quoted text (max ~50 chars, ellipsis if longer)
- Dismissable with (x) button, which also clears the selection highlight
- Only one active selection at a time — new selection replaces the previous one

### Agent Modification Behavior

- When a message includes a quoted selection, the Agent receives:
  - The full text of the containing entity (card content, scene description, etc.)
  - The exact selected substring
  - The user's instruction
- Agent returns:
  - A chat reply describing what was changed
  - The **modified full text** of the containing entity with only the selected portion changed
- Frontend applies the returned full text back to the source entity
- If the user expresses approval ("I like this phrasing"), no text change is needed — only record the preference

### Style Learning Integration

- Each inline correction triggers a `semanticAnnotation` via the existing pipeline
- The annotation's `factualChanges` records: source entity, original text, modified text, user instruction
- The annotation's `inferredPreferences` is generated by the existing LLM annotation service, now with richer per-word-level signal
- These annotations mix into the same pool as whole-state diff annotations and are injected into the Agent system prompt via `formatEditContextBlock`

## Scope Boundaries

### In scope
- Text selection + quote block UI across cards, scripts, shots, and chat replies
- Precise single-selection modification via Agent
- Style learning through existing semanticAnnotation pipeline
- Agent chat reply showing before/after

### Deferred for later
- Multi-selection (selecting multiple separate passages at once)
- Visual diff / change history UI
- Cross-project global style preferences
- Undo/revert for inline edits

## Success Criteria

- User can select text in any content area and see it quoted in the chat input
- Agent modifies only the selected portion without altering surrounding content
- Corrections are recorded as semanticAnnotations and influence subsequent generation
- The flow feels as natural as "quoting a message" in a messaging app

## Key Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Interaction entry point | Left-side chat with quote block | Preserves conversation context; familiar "quote reply" pattern |
| Style learning storage | Reuse existing `semanticAnnotation` | Avoids new tables; corrections are just higher-precision diffs |
| Selection mechanism | Browser-native Selection API | Zero custom UI for selection itself; works across all text areas |
| Feedback visibility | Agent confirms "I see it" via quote block, not inline tooltip | Keeps all Agent communication in one place (chat) |

## Dependencies

- Existing `semanticAnnotation` pipeline (`server/services/semanticAnnotation.ts`)
- Existing `editContext` service (`server/services/editContext.ts`)
- Existing `formatEditContextBlock` in story agent (`server/archive/storyAgent.ts`)
- `contentEditable` elements in `StoryCardsBoard`, `ScriptViewer`, `ShotTable`
