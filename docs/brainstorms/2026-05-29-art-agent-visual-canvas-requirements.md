# 美术 Agent + 自由视觉画布（视觉锚）

## 摘要

给 Story Cards 改成**图文合一卡**（每张卡 = 文字 + 它的图，住在一起，不另开画布），并配一个**「美术 Agent」**：它对用户上传的图做**理性 + 情绪双重解读**、把图通过「提示词工程」**重新出图（图生图）**、在对话里**越改越贴用户的审美**；卡里的图是下游剧本/镜头出图的**「视觉锚」**——成片的风格乃至具体镜头图从这里长出来。卡与卡之间用**「会说话的轻连接」**只在情绪真转变处显示小标（非装饰箭头）。v1 把「图文合一卡 + 美术 Agent 引擎」作为一个完整体验落地。

## 问题（Problem Frame）

产品现在能从「故事文字」长出剧本 / 镜头 / 图，但用户**真实的视觉素材进不来**：参考图、自己拍的照片没有入口；也没有一个「先把视觉感觉对齐」的地方。普通人脑子里常常**先有画面或参考**，却无法把它喂进创作、并据此调出「对的感觉」。同时，创始人的**审美判断（护城河）目前没有一个可见、可玩、可累积的载体**——审美只活在创始人脑子里，没在产品里现出一个「脸」。

## 角色（Actors）

- **A1 创作者（普通人）**：带着参考图 / 自己的照片来，想先把「视觉感觉」立住，再长成影像。
- **A2 美术 Agent**：对图做理性 + 情绪双重解读，把图变提示词重新出图，在对话中学习用户审美。
- **A3 自由视觉画布**：承载参考图 + AI riff 的图，自由拖放 / 编辑；作为下游出图的视觉锚。
- **A4（现有）小酌 / 创作引擎**：画布的视觉锚喂养其下游剧本 / 镜头出图。

## 关键流程（Key Flows）

- **F1 喂图 → 双重分析 → riff → 落画布**：用户在故事阶段上传 / 拖入参考图 → 美术 Agent 给出「客观内容」+「情绪 / 审美解读」→ 图生图出新图 → 落进自由画布。
- **F2 对话精炼 → 学习**：Agent 问「你想要什么」→ 用户在改图中迭代 → Agent 在本项目内累积其审美偏好 → 后续 riff 越来越贴「对的感觉」。
- **F3 视觉锚 → 喂养下游**：画布上定下的视觉感觉，成为下游剧本 / 镜头出图的风格 / 参考输入，影响成片。

## 需求（Requirements）

- **R1** 美术 Agent 对上传图做两种分析：**理性分析**（图里客观有什么）+ **美术 / 情绪分析**（接住用户情绪、读出审美）。
- **R2** 把图通过「提示词工程」转成生成**另一张**图的输入（图生图）。
- **R3** 对话式精炼：Agent 问「你想要什么」，用户在改图过程中迭代。
- **R4** 学习：Agent 在**单个项目内**累积用户审美偏好，使后续 riff 越来越贴「对的感觉」。
- **R5** Story Cards 改为**图文合一卡**：每张卡 = 记忆文字 + 它的图（参考图 / AI riff）住在同一张卡里；卡片可重排（顺序决定剧本），**不另开独立画布栏**。卡与卡之间用**「会说话的轻连接」**：仅在情绪真的转变处显示一个小标（如「暖 → 怅然」），不画装饰性「下一张」箭头；平淡处不显、不硬造。（2026-05-29 修订，取代原"自由画布栏"。）
- **R6** 画布角色 = **视觉锚**：画布上的图（参考 + AI riff）喂养下游剧本 / 镜头出图，成片风格乃至具体镜头图从这里长出来。
- **R7** 入口在**故事阶段（Story Cards 这侧）**上传 / 拖入参考图，美术 Agent 在这侧工作。
- **R8** **复用现有**视觉分析 + fal 生图 + 图片输入，在其上加「情绪 / 美术解读层」和「自由画布」，不重造。
- **R9** v1 = 画布 + 美术 Agent 引擎打包成一个**完整体验**（可分阶段落地，但目标是完整的）。

## 验收示例（Acceptance Examples）

- **AE1（覆盖 R1、R2）**：用户上传一张参考图，Agent 给出「客观内容」+「情绪 / 审美解读」两段，并 riff 出一张新图落进画布。
- **AE2（覆盖 R3、R4）**：用户说「再暖一点 / 更孤独一点」，Agent 重出；第二次 riff 已带上之前累积的偏好。
- **AE3（覆盖 R5）**：画布上多张图可自由拖动、各自修改，位置随意，不与文字卡对齐。
- **AE4（覆盖 R6）**：画布上定下的视觉感觉，反映在下游镜头出图的风格上（不是孤立摆设）。

## 成功标准（Success Criteria）

- 用户能把自己的参考图 / 照片喂进来，并在几轮内调出「对的感觉」。
- 画布上的视觉锚**真的影响**下游成片风格，而不是一个孤立的灵感角。
- 创始人的审美层第一次有了**可见、可玩、可累积**的载体。

## 范围边界（Scope Boundaries）

- 跨项目 / 全局的审美记忆持久化（v1 仅项目内）。
- 手机端这套（手机端正在单独重设；先在现有桌面 Story Cards 侧做）。
- PS 级图层 / 蒙版精修（v1 是「自由拖动 + 让 AI 改图」，不做专业修图）。
- 接入 fal 之外的新模型；视频生成。

## 关键决策（Key Decisions）

- **画布 = 视觉锚（喂养下游），非孤立灵感角。** 理由：让用户的视觉素材真正驱动成片风格，且与创始人「审美层」护城河咬合。
- **「学习」限本项目内。** 理由：先验证价值；跨项目持久化是后续。
- **复用现有视觉 / 生图管线，只加「情绪解读层 + 自由画布」。** 理由：不重造，聚焦真正新增的价值（审美解读 + 可玩画布）。
- **落地顺序（去风险）**：先做「上传图 → 双重分析 → 图生图 → 落进图文合一卡」主回路（最能验证价值），再上「对话精炼 / 学习」与「喂养下游」的强耦合。体验仍是完整的，只是先做最能验证的核心。
- **Story Cards = 图文合一卡，不另开画布（2026-05-29 修订）。** 理由：单开"自由画布栏"被判定"太复杂"；图住进卡里更简、心智单一。卡间用「会说话的轻连接」——只在情绪真转变处显示小标（如「暖 → 怅然」），不画装饰性"下一张"箭头（否则像流程图、且与"简化"相悖）。

## 依赖与假设（Dependencies / Assumptions）

- 依赖现有：`server/archive/visionAgent.ts`（视觉分析）、fal 生图、图片输入、`StoryCardsBoard`、Story / Shot 数据层。
- 假设：fal 的**图生图（image-to-image）**在现有管线可用 / 可接。
- 假设：「视觉锚喂养下游」的具体接线（画布图如何成为下游出图的风格 / 参考输入），在 planning 阶段细化。

- 画布的持久化，与 Story / 项目数据的关系（画布图存哪、怎么和文字卡关联）。
- 「学习用户审美」的具体表示，以及如何用它去 bias 后续 riff。
