# 意图识别 · 求职影片系统 — 需求对齐

## 总结

用户在故事聊天框丢简历 → 小酌认出「求职」意图、切换成「HR + 市场分析员」人设 → 深访挖出可信的职场优势 → 产出一份**可改的「优势清单」** → 交给现有「导演」(`synthesizeShotList`) 按**竞争力弧线**排序成一部三段式 showreel；影片视觉通过「风格词」走 art 资产库的「专业感」。

目标:生成一部**好看、有说服力、有市场竞争力**的求职影片。整条链路最大化**复用**现有系统(意图关、导演、出图管线、art 库),新建代码集中在「求职专列」,与 art repository 那条 worktree **解耦、不抢车道**。

---

## 背景与定位

- 今天(2026-06-15)已实现并合 main 的「意图确认关」只认 purpose 标签(求职/领英 是其一)。本需求把它从「认标签」**升级**成「认出后变成领域专家、产出量身定制的产物」。
- 产品平时主轴是「**情绪的流动**」;求职片里主轴**换成「职场竞争力」**,情绪/故事降为「给硬实力背书、加可信度」的辅助层。
- 与 art repository(99 图美术参考库,正由另一条 worktree 建设)的连接:**求职意图决定影片视觉调性**(详见「接口与车道划分」)。

---

## 钉死的需求（R1–R7，逐条确认）

- **R1 触发** — 故事聊天框丢简历 → 小酌从简历内容认出「求职」意图。复用现有意图关。*(摄取格式见开放问题,v1 先支持粘贴文字 + 文件上传。)*
- **R2 换人设 + 换主轴** — 认出求职后,小酌从「陪聊朋友」切成「HR + 市场分析员」;流水线主轴 情绪流动 → 职场竞争力。
- **R3 深访 + 优势清单** — 小酌像真 HR 追问简历里单薄/没量化处 → 挖出可信细节 → 产出**可确认/可改的「优势清单」**(= 求职版 ShotTable,简历 ↔ 影片之间的桥)→ 用户拍板。
- **R4 片子主体** — 硬实力/项目成果**可视化为主**(权重高);职业故事退为可信度背书。
- **R5 视觉走 art 库** — 意图(求职)产出一串「风格词」(专业/商务/可信/清晰质感),塞进 art repo 现成的 `findBestMatches(userDescription)` 入参。**art repo 不改,两条 worktree 解耦。**
- **R6 骨架** — 三段式 showreel:开场定位 → 中段优势逐个可视化(主体最长)→ 收尾「我在找什么 / 能带来什么」。默认骨架,小酌按简历微调、用户全程可改。
- **R7 导演层** — 复用现有 `synthesizeShotList` 排序 + 定景别/beat,弧线 情感 → 竞争力;**1:1**(一优势一镜),「展示形式」丰富度留 v2;导演的排布是**可改产物**,用户与导演协同改(勾出来改 + reflow)。

## 增强建议（AI 提，待岱岱拍板）

`◎` = 建议纳入 v1　`○` = 记一笔（v1 之后/可选）

- **◎ S1 照岗位/JD 反向定制（最大杠杆）** — 可选输入「目标岗位 / JD」。给了就做「岗位 vs 你」差距分析,把 showreel 重排成「为什么我适合**这个**」;没给则通用强项优先。同一批优势,目标不同、排序就不同——这是「市场竞争力」的核心动作。
- **◎ S2 说服弧线:钩子 → 证据 → 落点** — 把 R6 三段式拧成说服结构:开场甩**最能拉开差距的那个数字/成果**当钩子(抓住前 7 秒);中段每条优势 = 主张 + 量化 + 可视化证据;收尾 = 「我能为你带来什么 + 下一步」。
- **◎ S3 可信度三件套 + 忠于简历护栏** — 每条优势镜头带:能力点 + **量化结果** + 可视化素材。深访的真正任务 = 逼出缺失的数字。每个主张都**可追溯**回简历/深访(复用导演现成的 `sourceCardContent 原样回填`)。护栏:可框可强调,**不可杜撰**——夸大会让招聘者直接出局。
- **○ S4 一句「定位句 / thesis」当全片论点** — 市场分析员先找出用户的「不公平优势」(如 美术+电影+CS 的稀有交集),全片为这一句服务。复合背景尤其吃这套。
- **○ S5 平台/长度当硬约束** — LinkedIn/猎头片要短(30–60s)、前置高光、静音自动播(关键数字用文字叠层)。目标时长喂给导演 → 据此砍镜头数、逼优先级。
- **○ S6 反「AI 廉价感」美术护栏** — 求职片最忌看着像 AI 糊的。风格词偏「克制、真实、编辑感」,显式**负向排除**梦幻/AI 味——直接写进给 art 库的风格词。

## 「优势清单」数据结构（建议草案）

**顶层**
- `thesis` 定位句(S4)
- `targetRole` 目标岗位 / JD(可选,S1)
- `styleKeywords` 风格词(注入 art 库,R5 + S6)

**每条优势 `StrengthItem`**
- `claim` 能力点 / 主张
- `metric` 量化结果（深访重点逼出，S3）
- `visualCue` 可视化素材线索（怎么把它拍出来）
- `evidenceStory` 一句背书故事（R4 辅助层）
- `sourceRef` 来源引用（追溯回简历/深访，S3 护栏）
- `weight` 权重（导演排序用，影响是否入选/靠前）
- `presentationForm` 展示形式 *(v2 预留，v1 不用)*

→ 这份清单同时是 UI 上**可勾选编辑**的对象,也是喂给导演 `synthesizeShotList` 的输入(经适配层)。

## 架构落点

- **挂载点**:意图关「**求职已确认**」事件 → 激活「求职专列」(切人设 → 深访 → 优势清单 → 导演 → showreel)。
- **复用不碰**(别人的/共享的车道):
  - 意图识别/关(`recognizeStoryIntent` + StoryIntentGate)
  - 导演 `synthesizeShotList`([server/archive/shotSynthesis.ts](../../server/archive/shotSynthesis.ts))
  - 出图管线 `generateForMobile`
  - art 库 `findBestMatches`([server/services/artReferenceAgent.ts](../../server/services/artReferenceAgent.ts))
- **新建(本车道 5 块)**:
  1. 简历摄取(聊天框接收文件/文本)
  2. HR 深访 agent(追问 → 逼出量化)
  3. 优势清单 UI + state(可确认/可改)
  4. **适配层**:优势清单 → 导演卡片(`ShotListCardInput`) + 风格词 + 说服弧线上下文
  5. 求职专列触发(挂在「求职已确认」上)
- **弧线切换**优先走现有 `resonanceContext / confirmedIntent` 注入通道(意图关已在用),把「按竞争力/说服力排」喂进导演,**尽量不改导演 prompt**;真要改,也是加分支、不动情感片老路径。

## 接口与车道划分（与 art repository）

- **唯一接触点** = art repo 的 `findBestMatches(userDescription)` 入参。本车道**只读不改**,只把风格词注入到喂给它的描述里。
- **车道**:art-repo 那条管「镜头描述 → 长什么样」;本车道管「拍什么 + 往专业感锁」。**互不改对方文件。**
- **git 现状(2026-06-15 核实)**:主 checkout 当前停在 `feat/art-repository-system`,领先 main **6 个提交**,art 库代码**尚未并入 main**;全仓**只有一个 worktree**(art 库会话与本会话共用)。
- **做法**:本系统从 **main** 拉**独立 worktree** `feat/job-search-film-intent`,物理隔离;两条各自小步快合;**不同时做跨分支收敛**(AGENTS.md §5);worktree 内**不跑 dev server**(AGENTS.md §2),验证回主 checkout 的 3000。
