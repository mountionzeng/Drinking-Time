# 统一提示词片段池：图像提取的提示词 + 文字提取的提示词放在一起

## 摘要

把**图像分析提取的提示词**和**文字提取的提示词**统一成一个**项目级「提示词片段池」**——每个片段带来源（文字/图像）、标签（风格/色彩/构图/情绪/主体）、出处（卡/镜头）。重写 Shot Production Table 为「**池子 + 每镜引用**」：镜头从池里挑片段，经现有**合成缝**（`shotPromptComposer`）组装成出图 prompt。并在卡/镜头处做**两种提醒**：偏好驱动（「你常喜欢 X」）+ 缺口驱动（「这镜还缺视觉锚，从池里挑这些？」）。底层**复用美术 Agent 现有图像分析**，不加新模型；守则照旧（流动≠戏剧化、如实镜像、不翻负、不正向失真）。本轮**不做**对话框联想（留下一轮），但池结构为它铺路。

## 问题（Problem Frame）

现在图像分析的产出（美术 Agent 的「客观内容」+「美术/情绪解读」）在出图这一步只被压成 `shotPromptComposer` 里的**一行「视觉锚：…」**，和文字派生的提示词不是平起平坐：

- 图像提示词**不可复用、不可见、不可单独编辑**，更不能跨镜/跨卡再用；
- 用户喜欢的视觉点（多轮 riff 累积的审美）**没被主动记住并提醒**；
- 某镜/某卡**缺视觉锚时没人提示补**；
- 结果：图像这层的价值大量流失，用户每次重述「我想要的感觉」。

## 角色（Actors）

- **A1 创作者**：带图来，想让自己喜欢的视觉感觉**被记住、可复用**，而不是每次重说。
- **A2 美术 Agent**：把图像分析**提取成结构化、带标签的提示词片段**（复用现有双重分析，不换模型）。
- **A3 提示词片段池**：项目级单一真相源，承载**文字 + 图像两源**片段，可检索/排序/复用。
- **A4 Shot Production Table**：从池里**引用**片段组成每镜 prompt（后端 `shots` 表的真映射）。
- **A5 提醒逻辑**：偏好驱动 + 缺口驱动，在卡/镜头处主动提示。

## 关键流程（Key Flows）

- **F1 图像 → 片段**：用户喂图 → 美术 Agent 双重分析 → **提取成带标签的提示词片段**（风格/色彩/构图/情绪/主体）→ 进池，来源=图像、出处=该卡/镜头。
- **F2 文字 → 片段**：现有文字派生的提示词部分（主体/动作/台词/情绪电荷…）也表示为池里的片段，来源=文字。两源**平起平坐、同池**。
- **F3 表 = 池 + 引用**：重写 Shot Production Table——展示池子 + 每镜引用了哪些片段；镜头最终 `promptDraft` 由「引用的片段」经合成缝组装（不再是压扁的一行视觉锚）。
- **F4 偏好驱动提醒**：从累积偏好（他 riff/喜欢的图像片段、`visualPreference`）里，在卡/对话主动提醒「你常喜欢 X，要用上吗」。
- **F5 缺口驱动提醒**：某镜/某卡没有视觉片段时，提示「这镜需要 X，从池里挑这些？」并给候选。

## 需求（Requirements）

- **R1 图像分析片段化**：图像分析要产出**结构化、带标签、可复用**的提示词片段，而非只给散文解读。**复用美术 Agent 现有分析**，不加新模型。
- **R2 统一片段池**（项目级）：片段 = `{文本, 来源:文字/图像, 标签:风格/色彩/构图/情绪/主体, 出处:卡/镜头}`。文字派生与图像派生**平起平坐、同池**。
- **R3 重写 Shot Production Table = 池 + 引用**：镜头从池里引用片段；最终 prompt 由引用片段经**合成缝**（`shotPromptComposer` 仍是唯一组装点）组装。
- **R4 偏好驱动提醒**：把累积的图像偏好片段，在卡/对话主动提醒「你常喜欢 X」。
- **R5 缺口驱动提醒**：镜头/卡缺视觉片段时，提示从池里挑合适片段补上，并给候选。
- **R6 守则**：图像片段**如实来自分析、不编造**；提醒只反映**真实**累积偏好 / **真实**缺口，不硬推、不制造、不正向失真（与负面偏置同罪）。
- **R7 复用不重造**：美术 Agent 双重分析 / fal / 情绪管线 / `cardId` 关联 / `visualPreference` 全部复用。
- **R8 不破坏现有**：Story 出卡、`generateScript` 出镜头、视觉锚喂下游（B5）、合成缝的**文字层 + 情绪层**、自动存档都不能坏；无 `DATABASE_URL` 的 local-persist 降级不能坏。
- **R9 为 (b) 预留**：本轮不做对话框联想，但池结构必须让片段**可被检索/排序/从对话调用**，使下一轮 (b) 能直接从池子蹦。

## 验收示例（Acceptance Examples）

- **AE1（覆盖 R1、R2）**：喂一张暖色调参考图 → 池里出现该图的标签片段（如 `色彩:暖橙`、`情绪:温柔`、`构图:近景`），来源=图像、出处=该卡。
- **AE2（覆盖 R3）**：某镜引用了 2 个图像片段 + 文字片段 → 它的 `promptDraft` 里这些片段都**可见、可单独增删**，出图 prompt 由它们组装，**不再是压扁的一行视觉锚**。
- **AE3（覆盖 R4）**：用户多次 riff 偏暖、偏近景 → 在新卡/对话里被提醒「你常喜欢暖色调/近景，要用上吗」。
- **AE4（覆盖 R5）**：某镜没有任何视觉片段 → 表/卡上提示「这镜还没视觉提示词，从池里挑？」并给候选。
- **AE5（覆盖 R6）**：平淡故事 / 无真实偏好积累 → **不硬造提醒、不无中生有标签**；空就空着。

## 成功标准（Success Criteria）

- 图像价值不再流失：分析 → **可复用的标签片段**，进同一个池。
- 两源提示词**平起平坐、可见可编辑**，镜头 prompt 由引用片段组装。
- 用户喜欢的视觉点**被记住并主动复用**；某镜缺锚有提示。
- 为下一轮 (b) 对话框联想**铺好池**。

## 范围边界（Scope Boundaries）

- 本轮**不做** (b) 对话框联想（下一轮；池结构已预留）。
- 不加新图像模型、不换 fal。
- 跨项目偏好持久化（v1 仅项目内）。
- 手机端不碰。
- 不做 PS 级修图 / 蒙版精修。

## 关键决策（Key Decisions）

- **片段池（而非「每镜双栏」/「卡级档案」）**。理由：为 (b) 对话框联想铺路，片段可**跨镜跨卡复用、可检索排序**。（岱岱在 brainstorm 中明确选「片段池 + 引用」）
- **复用美术 Agent 分析、不加新模型**。理由：守则「不重造」；分析引擎已对（客观 + 美术/情绪）。
- **合成缝 `shotPromptComposer` 仍是唯一组装点**；本轮**演进它的「视觉层」**（一行视觉锚 → 引用的离散片段），但**保留文字层 + 情绪层 + 守则不变**。
- **两种提醒都做**（偏好驱动 + 缺口驱动）。
- **本轮产出是 spec（逻辑 + 数据要求）**；实现交给「另一个 Claude（终端跑）」，岱岱审核后合并 main。

## 依赖与假设（Dependencies / Assumptions）

- 依赖现有：`server/services/artAgent.ts`（双重分析 + riff + `visualPreference`）、`server/services/shotPromptComposer.ts`（合成缝）、`VisualCanvasItem`（`cardId` 已关联卡）、Shot Production Table（后端 `shots` 真映射）、`storyUpsert` 的 `body` **透传未知字段**（`z.record(...).optional()`，新增池字段**零迁移**）。
- 假设：美术 Agent 的分析可加一个「**结构化标签提取**」产出（LLM 结构化输出），**不换底层模型**。
- 假设：片段池可挂在 **story body JSON**（像 `visualCanvasItems` / `visualPreference` 那样持久化，localStorage + 服务器都走 `normalize*`），零或极小迁移。
- 持久化**双模式**（MySQL / local-persist）都要 round-trip 池字段（`normalize*` 要读出池）。

## 待解问题（Outstanding Questions，留给 planning / 实现）

- 图像 → 标签片段的提取：在美术 Agent 分析时**同步产出**结构化标签，还是**单独一步**？标签集合（风格/色彩/构图/情绪/主体）够不够、要不要可扩展？
- 文字派生片段：从 shot 字段/情绪电荷**自动切成片段**，还是保留现有文字层、只把**图像层**片段化并列？（影响「两源平起平坐」的实现强度）
- 片段 → `promptDraft` 的**组装顺序/权重**（合成缝里图像片段与文字层/情绪层怎么排）。
- 偏好驱动提醒的「常喜欢」如何判定（`visualPreference` 累积到什么程度算「常喜欢」）。
- 缺口驱动提醒的**候选排序**（从池里挑哪些片段给某镜——按标签匹配？按情绪匹配？）。
- 池的**去重/冲突**（同义片段、矛盾片段怎么处理）。
