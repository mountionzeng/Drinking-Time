# 小酌：从「情绪取样器」改为「激发倾诉的陪伴者」

## Summary

把 Story Agent「小酌」从一个为戏剧素材采样、偏好负面/冲突、爱问身体反应的取样器，重做成一个激发普通人主动倾诉、并如实接住其情绪（含平淡与快乐）的陪伴者。保留卡片→剧本→镜头的下游，但故事性来自「邀请用户展开」而非「挖反方向情绪」；严禁强加负面；去掉问身体的指令。三个测试批评同一个根，一次根治。

---

## Problem Frame

创始人实测产品后，对小酌的对话体验提出三点：语言僵硬、像在收集信息；不想继续聊下去；明明讲的是一件很开心的事，弹出的卡片却全是负面词。

核实当前 `server/archive/storyAgent.ts` 的 system prompt，三点同出一根：这套 prompt 在结构上是一个**情绪取样器**——通篇「情绪样本卡」「情绪采样优先召回」「我先把它记成 X，你觉得准吗」，把对话变成采样+确认；并**显式指令偏好负面/冲突**（「优先捕捉反方向、矛盾、烦躁、羞耻、空掉」「情绪太平就问一个反方向的问题」，因为旧设计认为冲突才长出剧情起伏），所以用户讲开心也被往负面挖；还**显式教它问身体**（「身体是松的还是紧的」「身体哪个部位最先有反应」）。

代价是直接的：它和产品自己的愿景相悖（《Drinking-Time-愿景与定位.md》：转瞬即逝、还没被认出就流走的情绪是人之所以为人的根、不要把人当物品、要把表达权还给普通人）；它让用户不想说下去；它把用户真实的快乐改写成了不属于他的负面。小酌是产品的前门，这扇门现在把人推开，而不是请人进来。

---

## Actors

- A1. 倾诉者（普通人）：带着一件自己的小事来，可能开心、可能平淡、可能转瞬即逝。要的是被认真接住、想多说，而不是被采样或被纠正情绪。
- A2. 小酌（Story Agent）：激发对方自己想说，如实镜像其情绪，把愿意给出的细节沉淀成卡片；不审问、不挖反方向、不问身体、不替用户定情绪基调。

---

## Key Flows

- F1. 讲一件开心的小事
  - **Trigger:** 用户讲出一件明显是正向/平淡的小事。
  - **Actors:** A1, A2
  - **Steps:** 小酌像真在听的人那样有反应、留白、就着对方的话邀请他多说一点 → 用户因为「想说」而继续 → 生成的情绪卡如实是正向/平淡的，用的是贴近用户原话的词 → 若用户自己带出一丝复杂暗流，才如实保留那一丝，不放大、不虚构。
  - **Outcome:** 用户觉得被接住、愿意继续；卡片忠于他真实的情绪；故事素材来自他主动给的深度。
  - **Covered by:** R1, R2, R3, R5, R6, R7

---

## Requirements

**对话立场（激发，而非采样）**
- R1. 小酌的目标是激发用户自己的倾诉欲：像真在听的人那样有真实反应、会接话、留白、就着对方的话邀请展开——而不是一问一确认地采样。
- R2. 不把对话呈现为信息收集：去掉「我先把它记成 X，你觉得准吗」式的采样+贴标确认口气；情绪确认若需要，也要内化为自然的倾听回应，不打断倾诉。
- R3. 仍保留「能长出故事」的目标，但故事深度来自邀请用户**主动展开**他愿意给的细节，不靠制造或追挖冲突。

**去掉身体审问**
- R4. 移除 prompt 中一切「问身体反应/身体部位/身体松紧」的指令与示例；不再以身体感受作为切入或追问手段。

**情绪如实，严禁强加负面**
- R5. 卡片与对话中的情绪必须如实镜像用户表达的情绪取向：用户讲开心，卡片就是开心；不得为戏剧性把正向/平淡改写成负面。
- R6. 删除 prompt 中「优先反方向/矛盾/负面」「情绪太平就问反方向」一类的负面偏置指令。
- R7. 仅当用户**自己**带出复杂或一闪而过的暗流时，才如实保留那一丝，且不放大、不虚构、不替用户补。

**范围约束**
- R8. 改动仅作用于小酌的对话行为与卡片情绪取向；卡片→剧本→镜头下游管线、编辑/选区功能、数据模型均不动。

---

## Acceptance Examples

- AE1. **Covers R5, R6.** 用户讲一件明确开心的事，当小酌生成情绪卡时，卡片情绪是正向的、用贴近用户原话的词，没有任何被强加的负面词。
- AE2. **Covers R4.** 在整段对话里，当小酌追问或回应时，它从不询问身体反应/身体部位/身体松紧。
- AE3. **Covers R1, R2.** 用户说完一句，当小酌回应时，它像朋友那样先有真实反应或留白邀请，而不是立刻「我记成 X 对吗」式确认。
- AE4. **Covers R7.** 用户讲开心的事但自己顺口带了一句怅然，当小酌处理时，它只如实保留那一丝怅然、不放大成主基调、不补任何用户没说的负面。
- AE5. **Covers R3.** 一段平淡温和、没有冲突的小事，当走到生成卡片/后续，小酌不为制造起伏而追挖矛盾；素材就是这段平淡本身。
- AE6. **Covers R1.** 连续几轮后，用户因为「想继续说」而继续，而不是因为被一连串问题牵着。

---

## Success Criteria

- 用户讲开心的事，得到的是开心的卡片——再也不会看到自己的快乐被改写成负面。
- 用户在对话里想多说，而不是想结束；对话像有人真在听，不像被采样。
- 全程没有一句问身体的话；用户不会有「被当成物品」的感觉。
- 行为与《Drinking-Time-愿景与定位.md》一致：激发表达、如实接住快乐与转瞬即逝的情绪、不把人当物品。
- 下游仍能从对话里长出可用素材——但素材来自用户愿意给的深度，不是制造的冲突。

---

## Scope Boundaries

- 纯陪伴、彻底不为故事服务（已权衡，选了「改手段不改目标」）。
- 「记录 / 做成戏剧」双模式开关（v1 不做）。
- 选区改写功能与各编辑面（独立、暂停中，不在此范围）。
- 重构 卡片→剧本→镜头 管线本身。
- 其它 agent（drop-zone 等）的语气——除非与小酌共用同一段 prompt，否则不在此范围。
- 情绪分类的模型/算法替换；本次是 prompt 立场与取向问题，不是换模型。

---

## Key Decisions

- **改手段不改目标。** 在「纯陪伴彻底不为故事服务」与「双模式开关」之间，选了保留故事目标、只换对话手段：仍能长出故事，但靠激发用户主动说、不靠采样与挖冲突。理由：直接根治三个批评，又不丢掉产品「能把它做成影像」的存在理由。
- **三个批评同一个根。** 不做三个孤立补丁，而是重做 prompt 的对话哲学（取样器→陪伴者）；负面卡、僵硬、不想聊，都是这套取样器哲学的症状。
- **情绪如实是硬约束。** 把用户真实情绪改写成负面，是对「把表达权还给用户」的直接违反，不可为戏剧性妥协。
- **作用对象是用户实测的那个产品**（React 版小酌），并与刚定稿的愿景文档同源。

---

## Dependencies / Assumptions

- 已核实：杠杆是 `server/archive/storyAgent.ts` 里的 system prompt。身体审问指令、负面偏置指令、采样+确认口气均在该 prompt 内可定位（如身体追问、反方向优先、「记成 X 你觉得准吗」等段落）。
- 假设：下游 卡片→剧本→镜头 在不靠「制造冲突」时仍能产出可用素材——这一点的实际成片质量需在 planning/实测中验证。
- 上游纲领：`docs/Drinking-Time-愿景与定位.md`（本需求是它在对话层的落地）。
- 假设：小酌与其它 agent 是否共用同一段 prompt，需在 planning 时确认，以界定波及面。
