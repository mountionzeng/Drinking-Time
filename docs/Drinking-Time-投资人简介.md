# Drinking Time · 投资人简介

> 一个面向影视视觉开发的双引擎平台：一端把参考与剧本分析成结构化生产输入，另一端把它转成可控的 AI 图像 / 视频结果。

## 一句话

通用 AI 视频工具解决的是"按下生成"。影视客户真正付钱的，是**有没有理解需求、能不能把模糊描述变成稳定可控的方案**。Drinking Time 做的就是这中间被跳过的一层。

## 问题

AI 生成模型已经足够强，但专业影视/广告/短片的视觉开发并没有因此变顺，原因有三：

- **理解断层**：客户给的是参考图、剧本、brief、零散笔记；模型要的是结构化、精确的视觉语言。中间这步翻译，今天靠人肉、不可复用。
- **控制缺失**：创作是反复分叉、比较、局部修正的过程。通用工具只会"整段重生成"——你只想改一句，结果原来对的部分也被毁掉。
- **资产不沉淀**：每个项目都从零开始 prompt，经验无法积累成可复用资产或壁垒。

## 为什么是现在

生成模型能力曲线已越过"能用"，竞争正从"谁能生成"转向"谁能稳定地生成对的东西"。能把行业理解结构化、并把它变成跨模型可复用资产的人，开始拥有模型之外的护城河——而模型本身正在快速商品化。

## 解法与产品

双引擎，结构化中间层是核心资产：

- **Analysis Engine**：碎片输入 → 自动识别/按时间轴整理/加权 → 沉淀为可跨项目复用、可人工继续编辑的**环境模板**与**镜头蓝图**。prompt 只是模板的派生物，模板才是资产。
- **Creation Engine**：结构化结果 → 节点式编排 → 生成、批量试错、版本回溯。关键能力是**指针/选区编辑**：指到某一处，只改那一处，保留其余——localized edit，而非 full regenerate。

两条进入路径（拖素材的"素材路"、跟 Agent 聊出来的"故事路"）最终都收敛到同一张可生产的镜头表。

## 壁垒

不是模型，模型在贬值。护城河是三件事的组合：

1. **分析能力 + 审美判断 + 生成控制**：创始团队具备传统影视数字绘景背景，能读懂剧本与镜头语言，把"感觉"翻译成"可执行视觉参数"——这是数据集学不来、API 调不出的部分。
2. **结构化中间层**：输入不直接拼成一句 prompt，而是沉淀成模板。带来模型可替换性、跨模型对比能力、案例库与方法论积累。
3. **可复用环境模板库**：接单越多，资产越厚，边际成本越低——形成随时间加深的系统壁垒。

## 现在做到哪（诚实陈述）

当前阶段：**创始人自用 + 原型验证**，本地可跑通端到端。已落地的具体能力：

- 故事驱动工作流跑通：跟 Agent 对话 → 故事卡片 → 合成剧本（logline/主题/情感弧线/分场）→ 落成镜头表。
- 全链路**内联可编辑 + 持久化**已上线：故事卡片、剧本、镜头表的内容均可直接改、即时存（本地 + 服务端）。
- **选区定向编辑**（指到一段、只改一段、给对比、确认才替换）已完成需求定义、进入实现。
- "用户改过的就是用户的、Agent 重生成不覆盖"的不覆写契约已设计，正逐面接入。

尚未做（不回避）：规模化 SaaS、真实视频模型批量接入、跨项目模板库的产品化、协作权限层（用户自带 Agent 的受控协作插槽已有接口设计，未实现）。

## 差异化

- **vs 通用 AI 视频工具**：它们止于"会生成"；我们值钱在生成之前的结构化理解层，和生成之中的定向局部控制。
- **vs 节点类生成工具（ComfyUI 等）**：那是面向参数玩家的重工具；我们面向影视创作者的工作流，节点是手段不是门槛，叙事与镜头语言是一等公民。

## 商业

面向影视/广告/短片客户的专业工作台。客户买的不是按钮，是"把参考资料转成高质量结构化生产输入"的那一层，以及更快给出概念方案与样片的能力。第一版不做完整 SaaS，先做"专业展示 + 可操作原型"，验证流程与付费意愿。

## 风险与下一步

- 主要风险：模型能力跃迁可能压缩部分中间层价值——对冲在于结构化层本身与影视方法论资产，不依赖单一模型。
- 下一步：完成选区定向编辑与不覆写契约的全面落地 → 接真实生成模型 → 把环境模板库产品化为可复用资产 → 从创始人自用走向小范围客户验证。

## 一句话总结

模型在变成水电，会用模型不再稀缺。稀缺的是把行业理解结构化、把控制权还给创作者、并让这套能力随项目越积越厚的那一层——这就是 Drinking Time。
