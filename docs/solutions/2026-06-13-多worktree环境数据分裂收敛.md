# 多 worktree 环境数据分裂与一次性收敛

## 问题

vibe coding 过程中积累了 6 个 worktree、3+ 个并行 dev server，每个端口的数据互相对不上。
表象是"不知道有几个环境、数据在哪"，根因在持久化机制：

- `server/db.ts` 的本地数据路径 `DEFAULT_LOCAL_PERSIST_PATH = path.join(process.cwd(), ".webdev", "local-persist.json")` **跟启动目录（cwd）走**。
- 每个 worktree 启动的 dev server 读写各自目录下的 `.webdev/local-persist.json`。
- 所以"多 dev server 并行" = "数据分裂成多份互不相通的副本"，且各副本 id 空间冲突（同 id 不同内容）。
- 图片不受影响：在共享目录 `.webdev/images`（`.env` 的 `LOCAL_IMAGE_DIR`），所有环境同源。

历史伤疤：2026-06-01 测试把真实数据原子覆盖一次（`server/db.ts` 已加测试防误写 + `.webdev/backups/` 安全网）；2026-06-12 数据分裂成 6 份。

## 解决方案

一次性收敛到"主仓库单一环境、一份数据"，并立长期防线。分四类工作：

1. **环境仪表盘** `pnpm env:status`（`scripts/env-status.ts`）——实时输出 worktree / 端口 / 数据文件，多 dev server 并行时首行醒目警告。采集（`git worktree list --porcelain`、`lsof -nP -iTCP -sTCP:LISTEN`、按 pid 的 cwd 归属）与解析/格式化纯函数分离，纯函数单测。
2. **数据合并工具** `scripts/merge-local-persist.ts`——多份 local-persist.json 按内容哈希去重、全量 id 重编号、外键重映射（含自引用 editSnapshots、generatedImages 等）。默认 dry-run 出报告，`--write --out` 才落盘且拒绝覆盖源文件。
3. **AI 规则铁律** 根目录 `AGENTS.md` + `CLAUDE.md` 指针——约束后续 AI 会话：只主仓库跑 dev、worktree 用完即删、禁在 worktree 写数据。
4. **用户指南** `docs/environment-guide.md`——数据机制、端口约定、三步排查、事故史。

## 关键陷阱（踩过的坑）

- **dev server 有 `while-true` bash 守护**（`scripts/dev-mobile.sh` 形态 + 代理 export）。直接 `kill` 子进程会被守护立刻重启；必须先杀守护脚本本身，再杀子进程。孤儿进程会被 launchd 收养，每次 kill 后以新 pid 重现——要顺进程树找到真正的启动器。
- **`lsof` 必须带 `-nP`**：不带 `-P`，macOS 把端口解析成服务名（3000 → `hbci`），数字端口解析全落空。
- **静态文档会过期**：仪表盘选择"每次实时生成、不落盘快照"，这就是"应对变化的逻辑"。
- **合并不是挑一份**：每个环境都有独有故事（如最早的"听到鸟叫"系列只在主仓库旧备份里、"手机端回忆"只在 agent-layer）。必须按内容合并；同 createdAt 不同内容 = 分叉副本，工具只报告、由人拍板。
- **嵌套 worktree 归属要取最长前缀**：`.claude/worktrees/*` 在主仓库目录内部，按前缀匹配会被误归属到主仓库，需按路径长度降序匹配。
- **数据落位的时序炸弹**：放入合并数据后，若有**早于落位时刻启动**的旧 dev server，其内存里是旧数据，一旦存盘就覆盖合并结果。务必在收敛末尾**确认没有早启动的服务在跑**，必要时杀掉重启，让服务重新加载合并后的数据。用 `ps -o lstart` 比对启动时刻 vs 数据落位 mtime。
- **`git branch -d` 的合并检查针对当前 HEAD**：要删的分支若已并入 main 但未并入当前分支，`-d` 会报"not fully merged"。先 `checkout main` 再删。
- **切分支被未跟踪文件挡路**：未跟踪文件与目标分支的已跟踪文件同名会 abort。先 `cmp` 比对，一致则删未跟踪副本再切。

## 收尾标准动作（可复用清单）

1. 收割未完成的 merge / 未提交同步文件
2. 完成分支合并（功能源头分支合入收敛目标）
3. **全量备份**所有环境的 local-persist.json 到带日期的目录
4. 停掉所有 dev server（含 bash 守护），用 mtime 确认数据冻结
5. 用最新实时数据跑合并工具 dry-run，人工核对故事清单与分叉副本
6. `--write` 落盘到暂存文件，校验 JSON、id 唯一性、珍贵内容在列
7. 拆 worktree（`--force`）→ 删分支 → 主仓库切 main
8. 合并数据落位 `.webdev/local-persist.json`
9. **干净重启 dev**，经 API（如 `/api/archive/stories`）验证加载的故事数，确认文件未被覆盖
10. `pnpm env:status` 见绿（单一服务、无警告）
