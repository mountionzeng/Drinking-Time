# 美术参考库（Art Repository）

这是整个系统的美术审美控制中心。

## 目录结构

```
art-repository/
├── references/          # 99张精选参考图片（1024×1024）
├── rules/
│   └── global-rules.md  # 全局美术规则库
├── metadata.json        # 参考图片的元数据索引
└── README.md           # 本文件
```

## 核心概念

### 参考图片库 (`references/`)

你精心挑选的 99 张好看的图片。每张都已标准化到 1024×1024 的正方形，便于系统处理和展示。

**使用流程**：
1. 查看这些参考图
2. 在 `metadata.json` 中为每张图添加标签和描述
3. 系统在生成时会自动检索匹配的参考

### 美术规则库 (`rules/global-rules.md`)

你的美术审美原则的文字化表达。

**示例规则**：
- "色调倾向于暖色，避免冷冰冰的蓝调"
- "光线必须有明确的主光源，避免均匀平面光"
- "避免过度饱和的色彩"

这些规则会被编织到每次生成的提示词中。

### 元数据索引 (`metadata.json`)

每张参考图的标签和描述。结构如下：

```json
{
  "03594f4458acbae0dd26d5f300a29752.jpg": {
    "visualFeatures": ["构图", "光影"],
    "scenarioTags": ["室内", "静谧"],
    "mood": "安静沉思",
    "lightingStyle": "侧光",
    "colorTone": "暖色",
    "userNotes": "非常喜欢的光影处理"
  }
}
```

## 下一步

### 给参考图片打标签（优先级：高）

打开 `metadata.json`，为你的 99 张图片填充以下信息：

- **visualFeatures**: 这张图的主要视觉特征（构图、光影、色彩、质感等）
- **scenarioTags**: 适用的场景标签（室内、户外、静谧、明亮等）
- **mood**: 这张图的情感基调（安静、热烈、温馨等）
- **lightingStyle**: 光线类型（自然光、侧光、逆光、柔光等）
- **colorTone**: 色调（暖色、冷色、中性等）
- **userNotes**: 你的个人备注

**建议**：不需要一次性完成所有 99 张。可以先标记最重要的 20-30 张。

### 编写美术规则（优先级：高）

打开 `rules/global-rules.md`，写下你的美术原则。

**关键**：规则要具体、可执行。

### 系统集成（优先级：中）

完成上述两步后，系统会集成这个参考库到生成流程中：
- 生成图片时，自动从参考库提取相关指导
- 融合到提示词中
- 确保所有生成的图都符合你的审美标准

## 文件说明

- `art-repository/` 是一个完整的、独立的知识库。可以版本控制、备份、甚至导出给其他项目使用。
- 所有参考图片已优化为 1024×1024，避免了尺寸混乱带来的问题。
- 元数据和规则是纯文本/JSON，易于编辑和维护。
